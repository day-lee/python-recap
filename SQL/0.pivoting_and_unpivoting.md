### pivot table 
- 가로형 데이터는 사람이 읽기 편하다. 엑셀 보고서 처럼 컬럼이 옆으로 늘어나서 한눈에 분석이 쉽다. 
- 세로형 데이터는 DB 친화적이다. 가로로 새로운 컬럼을 추가할 필요 없이 아래로 행만 늘리면 된다. (행의 distict 값이 늘어난다.)
- 중심 고정축을 기준으로 용도에 따라 가로, 세로로 변환하여 사용한다. 
- CASE WHEN.. 문법으로 구현 가능

### Pivoting 
- 리포트 friendly
- rows to columns (가로로 확장)

### Unpivoting
- DB friendly 
- columns to rows (세로로 확장)

--------------------------

### 보고서 만들 때 유용한 group_concat
- group_concat([DISTINCT] 표현식 [ORDER BY 필드] [SEPARATOR] '구분자' )
e.g. group_concat(DISTINCT platform ORDER BY platform ASC SEPARATOR ',' )
- 위의 DB 데이터를 한칸에 합쳐서 보기 쉽게 만들 수 있다.
