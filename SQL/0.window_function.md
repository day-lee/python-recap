### 윈도우 함수
- 그룹바이는 행을 압축해버려서 (collapse) 다른 행 값을 상상해야하지만, 
  윈도우 함수는 기존 테이블의 행을 유지하면서 옆에 임시 데이터를 새 컬럼으로 덧붙여 준다. 
- partition by는 필수가 아니다. 그룹핑 말고 전체 데이터 기준으로 정렬, 번호 메기려면 OVER (ORDER BY col) 형태로 쓴다 
- 순위, 값 분석에는 ORDER BY가 필수다. 기준이 있어야 하니까. 

- 기존 데이터 다 날리고 요약본만 보고싶다면 group by
- 기존 상세 데이터와 함께 추가 정보를 붙이고 싶다면 window function 

### 윈도우 함수 자리 3가지 종류

#### 1. Ranking Functions 순위 결정
- ROW_NUMBER(): 중복 없이 1,2,3 순서대로 행 번호 메김
- RANK(): 동점자가 있으면 공동 순위 매김. 2등이 없음 (공동1등, 공동1등, "3등"...)
- DENSE_RANK(): 동점자가 있어도 순위 건너뛰지 않음 (공동1등, 공동1등, "2등"...)
- order by 기준 필수 

#### 2. Value Functions 값 분석 
- LAG(col): 현재 행보다 이전(위) 값을 가벼옴. e.g. 어제와 오늘 매출 비교 
- LEAD(col): 현재 행 다음(아래) 값 
- FIRST_VALUE(col), LAST_VALUE(col): 그룹 내 가장 첫번째나 마지막 값 
- order by 기준 필수 

#### 3. Aggregate Functions 집계 
- SUM(), AVG(), COUNT(), MAX(), MIN(): 행 압축시키지 않고 누적 합계(running, rolling total) 보여줌 
- 이전 + 현재 cumulative 값이라고 보면됨
