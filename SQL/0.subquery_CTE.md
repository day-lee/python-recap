# Subquery vs CTE(common table expression)
- 보통은 with 문으로 시작하는 CTE가 가독성, 재사용성이 좋다. 
- subquery와 cte는 윈도우 함수로 다시 쓸 수도 있다. 

## 서브쿼리 3 경우
### 1. 단일 값 비교 (scalar subquery) 
    SELECT log_id, sensor_id, value
    FROM sensor_logs
    WHERE value > (SELECT AVG(value) FROM sensor_logs);

### 2. SELECT절 서브쿼리 - 간단한 통계치 
    SELECT s.sensor_id, s.location, 
            (SELECT COUNT(*) FROM sensor_logs l where l.sensor_id = s.sensor_id) as total_logs 
    FROM london_sensors s; 
    
### 3. EXISTS 상호연관 서브쿼리 
- 메인 쿼리와 서브쿼리의 sensor_id가 실시간 매핑되어 더 직관적임 

    SELECT s.sensor_id, s.location FROM london_sensors s 
    WHERE EXISTS (
        SELECT 1 FROM sensor_logs l 
        WHERE l.sensor_id = s.sensor_id AND l.status = 'Critical')
