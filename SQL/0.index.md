EXPLAIN (예측치)
- 쿼리를 실제로 실행하지 않고, 데이터베이스 엔진(옵티마이저)의 계획만 보여줌
- 인덱스 유무만 알고 싶을 떄 
- 예측이라 틀릴 수도 있음 

type: const, eq_ref 나오면 됨. ALL은 인덱스 안탄거 
key: 인덱스의 이름 


EXPLAIN ANALYZE (실측치)
- 쿼리를 실제로 실행합니다. 실행하면서 각 단계별로 시간이 얼마나 걸렸고, 몇 개의 행(Row)을 읽었는지 실제 데이터(실측치)를 측정해서 보여줌. 
- 진짜 실행하는거라 정확함 
- 실제 걸린 시간, 읽은 행 수를 확인할 수 있음 
- insert, update, delete 쿼리는 운영 서버에서 실행하면 실제로 실행됨. 
- 트랙젝션 BEGIN; ... ROLLBACK; 으로 감싸서 실행하거나 테스트 서버에서 해야함
