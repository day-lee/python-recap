
### JOIN 시 WHERE or ON
- WHERE은 조건에 안맞으면 삭제함 -> 옛날 방식
- ON은 기준 테이블은 지키고, 데이터가 없으면 null 처리한다. -> 표준 
- inner join / outer join(left, right, outer)

### GROUP BY & Aggregate functions(SUM, AVG, MAX, MIN)
- group by는 특정 필드 기준으로 그루핑해서 방을 만들어 준다. 
- 그 방에는 한개 로우만 해당할 수도 있고, 여러 로우가 존재할 수도 있다.
- 만약 그룹바이에 기준을 주지 않은 필드를 셀렉트에서 쓰면, 여러 로우가 들어있는 그룹에서는 그 중에 어떤 값을 보여줘야 할지 정할수가 없다. 
- 따라서 셀렉트에 쓴 일반 필드는 꼭 그룹바이에도 써야한다.  
- group by는 피봇테이블과 같다. 
 
- select에 올 수 있는 두 가지 경우, group by  
    1) 어느 필드 던 집계함수(sum, avg..)로 압축된 대표 숫자 결과 
    2) 기준 필드 이름 - 그룹핑 된 방의 이름과 같다.

### Having은 aggregation 관련 필터링할 때 group by 뒤에서 사용한다. 

### COUNT(DISTINCT field)
- 중복 없이 개수 세기 

### ABS()
- 절대값. 주식 등락 상승 10%, 하락 10% 모든 경우를 포함하고 싶을 때 쓸 수 있음 

### division: decimal or floating-point, integer 처리 때문에 타입 정해주는 트릭 필요할 수 있음
- 값 * 1.0
- CAST()
CAST(10 AS DECIMAL)/4,
CAST(10 AS FLOAT)/4,
- ::DECIMAL/::FLOAT
10::DECIMAL/4,
10::FLOAT/4,

### 퍼센티지 %
0.50 과 50%는 같은 값이다. 0.50은 분석의 관점일 때, 50%는 보고서에 시각화 해서 보여줄 때 쓴다. 

### ROUND() 
0.50 to one decimal place -> 50.0

### NULL: missing vs pending, incomplete 
- 값이 누락됨 
- 아직 값이 계산되지 않음(테스트 결과가 아직 안나옴, 광고 효과 수익이 아직 계산되지 않음) 

### NULL handling 
- COALESCE(arg1, arg2, arg3, default) 여러 필드 중 처음으로 NULL 이 아닌 값을 사용하겠다. 모두 NULL 이면 default 값 사용한다. 
    용례: 임직원의 모바일, 랜드라인, 비상연락망 중 연락처 값이 있는 것으로 채워준다. 
- IFNULL(field, default) field 값 평가해서 NULL이면 디폴트 값 사용하겠다.
